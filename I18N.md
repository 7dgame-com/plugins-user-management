# 用户管理插件 - 多语言支持

## 概述

用户管理插件已实现完整的多语言支持，支持以下 5 种语言：

- **zh-CN**: 简体中文（默认）
- **zh-TW**: 繁体中文
- **en-US**: 英语
- **ja-JP**: 日语
- **th-TH**: 泰语

## 语言切换机制

插件通过 URL 参数自动获取主前端的语言设置：

```
http://localhost:5173/?lang=en-US&theme=light
```

主前端在加载插件时会自动传递 `lang` 参数，插件会根据该参数自动切换语言。

## 文件结构

```
src/i18n/
├── index.ts              # i18n 配置入口
└── locales/
    ├── zh-CN.ts          # 简体中文
    ├── zh-TW.ts          # 繁体中文
    ├── en-US.ts          # 英语
    ├── ja-JP.ts          # 日语
    └── th-TH.ts          # 泰语
```

## 语言包结构

每个语言文件包含以下模块：

### common - 通用文本
```typescript
common: {
  search: '搜索',
  add: '添加',
  edit: '编辑',
  delete: '删除',
  // ...
}
```

### user - 用户管理
```typescript
user: {
  title: '用户管理',
  username: '用户名',
  email: '邮箱',
  roles: {
    root: '超级管理员',
    admin: '管理员',
    // ...
  },
  messages: {
    fetchFailed: '获取用户列表失败',
    // ...
  }
}
```

### invitation - 邀请管理
```typescript
invitation: {
  title: '邀请管理',
  generate: '生成邀请链接',
  // ...
}
```

### register - 用户注册
```typescript
register: {
  title: '用户注册',
  username: '用户名',
  // ...
}
```

### layout - 布局相关
```typescript
layout: {
  waitingAuth: '等待主系统授权...',
  requireMainSystem: '请从主系统打开此插件',
}
```

## 在组件中使用

### 基本用法

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>

<template>
  <div>
    <h1>{{ t('user.title') }}</h1>
    <el-button>{{ t('common.add') }}</el-button>
  </div>
</template>
```

### 带参数的翻译

```vue
<template>
  <p>{{ t('invitation.registrationRecords') }} — {{ code }}</p>
</template>
```

### 在 JavaScript 中使用

```typescript
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

ElMessage.success(t('user.messages.createSuccess'))
ElMessage.error(t('user.messages.fetchFailed'))
```

## 添加新的翻译

1. 在 `src/i18n/locales/zh-CN.ts` 中添加中文翻译
2. 在其他语言文件中添加对应翻译
3. 在组件中使用 `t('key.path')` 引用

示例：

```typescript
// zh-CN.ts
export default {
  user: {
    newField: '新字段',
  }
}

// en-US.ts
export default {
  user: {
    newField: 'New Field',
  }
}
```

## 测试多语言

```

http://localhost:5173/?lang=ja-JP

```

### 在主前端中测试

1. 启动主前端：`cd web && pnpm dev`
2. 在主前端切换语言
3. 打开用户管理插件，插件会自动使用主前端的语言设置

## 注意事项

1. **默认语言**: 如果 URL 中没有 lang 参数，默认使用简体中文（zh-CN）
2. **回退机制**: 如果某个翻译键不存在，会自动回退到简体中文
3. **语言代码**: 必须使用完整的语言代码（如 zh-CN），不支持简写（如 zh）
4. **同步更新**: 添加新功能时，请同时更新所有 5 种语言的翻译文件

## 主前端集成

主前端通过 `PluginLayout.vue` 自动传递语言参数：

```typescript
const pluginUrl = computed(() => {
  const baseUrl = pluginManifest.value?.url ?? "";
  if (!baseUrl) return "";
  const separator = baseUrl.includes("?") ? "&" : "?";
  return `${baseUrl}${separator}lang=${appStore.language}&theme=${currentThemeName.value}`;
});
```

插件会在初始化时读取 URL 参数并设置语言：

```typescript
function getLanguageFromURL(): string {
  const params = new URLSearchParams(window.location.search)
  const lang = params.get('lang')
  if (lang && ['zh-CN', 'zh-TW', 'en-US', 'ja-JP', 'th-TH'].includes(lang)) {
    return lang
  }
  return 'zh-CN'
}
```

## 维护建议

1. 保持所有语言文件的结构一致
2. 使用有意义的键名，便于理解和维护
3. 对于长文本，考虑拆分成多个键
4. 定期检查是否有遗漏的翻译
5. 考虑使用翻译管理工具进行协作翻译
